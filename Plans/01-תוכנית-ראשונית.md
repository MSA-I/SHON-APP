# תוכנית פיתוח: אפליקציית ניהול לקוחות - שון בלאיש (מעצב אירועים)

## Context

המעצב **שון בלאיש** עובד היום בתהליך ידני: לקוח נכנס לפגישה, המעצב ממלא דף תכנון Word עם פרטי האירוע, ואז מדפדף איתו ידנית בתיקיות תמונות במחשב, בוחר עיצובים, ומסכם הכל ידנית. המטרה: להפוך את התהליך לאוטומטי, מהיר, מקצועי ומסודר - ולהעניק חוויית פגישה יוקרתית הראויה למעצב פרימיום.

## ארכיטקטורה

### עקרונות יסוד
1. **Local-First:** כל המידע נשמר במחשב המקומי. אין שרתים, אין login, אין אינטרנט נדרש.
2. **תמונות במקום:** האפליקציה קוראת ישירות מהתיקיות הקיימות ב-`D:/משה פרוייקטים/שון בלאיש/`.
3. **גיבוי ידני:** המעצב יוכל לייצא/לייבא את כל הנתונים כקובץ JSON אחד.
4. **RTL מלא:** עברית בלבד, RTL מקצה לקצה.

### Tech Stack (מתבסס על הטמפלייט הגלובלי)
- **Framework:** React 19 + TypeScript + Vite
- **Styling:** Tailwind (inline CDN, כמו בטמפלייט)
- **Animations:** Framer Motion (מעברים אלגנטיים בין מסכים, hover effects בגלריה)
- **Icons:** Lucide React
- **State:** React Context + useReducer (פשוט, ללא Redux)
- **Persistence:** **IndexedDB** דרך `idb` library (אחסון נתוני אירועים, favorites, חתימות)
- **תמונות:** **Electron** או **Tauri** עבור גישה ל-filesystem המקומי, או חלופה: Vite dev server מצביע ל-folder תמונות + העתקה לתיקיית `public/images/` (יידון בהמשך)
- **PDF Export:** `pdf-lib` + `@pdf-lib/fontkit` לתמיכה בעברית ועיצוב יוקרתי
- **חתימה דיגיטלית:** `react-signature-canvas`

### החלטה ארכיטקטונית: גישה לתיקיות התמונות
מכיוון שהתמונות נשארות במקום ולא מועלות לשרת, יש שתי אפשרויות (תיבחר בשלב הביצוע):

**אופציה A: Tauri / Electron app (מומלץ)**
- אפליקציה native עם גישה מלאה ל-filesystem
- מתקינים פעם אחת ופותחים כמו תוכנה רגילה
- מהיר, יציב, ללא Vite dev server

**אופציה B: Vite + Symbolic Link**
- Symlink מתיקיית הפרויקט אל תיקיית התמונות בכונן D
- האפליקציה נטענת מ-localhost:5173
- פשוט יותר לפיתוח, אבל פחות מקצועי לפריסה

## פיצ'רים מרכזיים

### 1. גלריית תמונות חכמה (הלב של האפליקציה)
- **סריקה אוטומטית** של תיקיות התמונות בעת טעינה
- **Tabs לקטגוריות**: אולם בסיס / חופות גאמוס / חופות ריזורט / חופות שדרוג / מפות מפיות / עיצובים שדרוג / ריזורט בסיס
- **חיפוש חופשי** לפי שם הקובץ (התמונות בעברית - "שנדליר ורסאצה", "פמוט דובאי")
- **Thumbnails נטענים lazy** עם blur placeholder
- **Lightbox** לתצוגה מלאה בלחיצה
- **בחירה מרובה** עם counter (1/5 selections)
- **Notes לכל תמונה נבחרת** (למשל: "בצבע זהב", "להגדיל ב-20%")
- **Smart sorting**: תמונות שנבחרו בעבר ע"י לקוחות אחרים מקבלות סימון "פופולרי"

### 2. PDF Export יוקרתי
- כותרת עם לוגו "שון בלאיש - מעצב אירועים" (placeholder עד שיתקבל לוגו)
- פרטי אירוע מלאים
- **תמונות נבחרות מובנות בקובץ** (כך שהלקוח יוכל לראות אותן גם אחר כך)
- סעיף שדרוגים מפורט
- חתימה דיגיטלית בתחתית
- תנאים משפטיים מהדף המקורי (העתקה ישירה מה-DOCX)
- עיצוב בהשראת invitations יוקרתיות - עברית RTL מלאה

### 3. ניהול לקוחות
- **רשימה עם חיפוש** (לפי שם, תאריך, או לוקיישן)
- **סטטוסים**: טיוטה / נחתם / הושלם
- **הצגת תאריך אירוע** עם הדגשה לאירועים בקרובים
- **שכפול לקוח** (Copy as template) - שימושי לזוגות עם דרישות דומות
- **ייצוא/ייבוא JSON** לגיבוי

### 4. חתימה דיגיטלית
- Canvas signature במסך הסיכום
- שמירת חתימה כ-base64 ב-IndexedDB
- הטמעה אוטומטית ב-PDF הסופי

### 5. Presentation Mode (Phase 2 - לא ב-MVP)
- מסך מלא יפה לפגישה, ללא navigation
- מעברים בין סקציות עם מקלדת/חצים
- מתאים להצגת התוכן ללקוח בצורה דרמטית

## שלבי פיתוח (MVP First)

### שלב 1: בסיס + תיקיות (1-2 ימים)
1. אתחול פרויקט Vite + React 19 + TypeScript
2. הקמת Tailwind inline + פלטת Luxury Editorial
3. ייבוא פונטים (Frank Ruhl Libre + Heebo)
4. רכיבי UI בסיסיים (Button, Input, Card)
5. RTL setup

### שלב 2: ניהול לקוחות (1-2 ימים)
1. IndexedDB setup עם `idb`
2. CRUD לקוחות
3. רשימה ראשית + חיפוש
4. טופס יצירת/עריכת לקוח

### שלב 3: טאבים של אירוע (2-3 ימים)
1. Layout עם sidebar וטאבים
2. טאב פרטי אירוע
3. טאב מפות ומפיות
4. טאב חופה
5. טאב שדרוגים

### שלב 4: גלריית תמונות (3-4 ימים) — **הלב של המוצר**
1. בחירת ארכיטקטורת קריאת תמונות (Tauri/Electron vs Symlink)
2. סריקת תיקיות וקריאת רשימת תמונות
3. Thumbnails generation או lazy loading
4. תצוגת רשת + פילטרים + חיפוש
5. Lightbox + בחירה מרובה
6. הוספת notes לתמונה נבחרת

### שלב 5: PDF + חתימה (2-3 ימים)
1. תצוגת סיכום preview
2. SignaturePad
3. יצירת PDF עם pdf-lib (כולל עברית RTL!)
4. הטמעת תמונות נבחרות ב-PDF

### שלב 6: ליטוש + גיבוי (1-2 ימים)
1. ייצוא/ייבוא JSON
2. אנימציות Framer Motion
3. Edge cases + שגיאות
4. בדיקה מקצה לקצה

**זמן פיתוח כולל משוער: ~3-4 שבועות**

## Verification (איך נדע שזה עובד)

### בדיקה ידנית מקצה לקצה
1. **יצירת לקוח חדש** → מילוי "ליאור ודן", 14.06.2026, 350 מוזמנים
2. **מילוי טאב פרטי אירוע** → לוקיישן ריזורט, שעה 20:00
3. **מילוי טאב מפות** → צבע פשתן, קיפול X
4. **גלריית עיצובי שולחן** → בחירת 3 תמונות עם הערות
5. **טאב חופה** → סוג מרובעת, מיקום בריכה, בחירת תמונת חופה
6. **שדרוגים** → הוספת טקסט חופשי
7. **סיכום** → פתיחת preview, חתימה, ייצוא PDF
8. **בדיקת PDF**: תמונות מוטמעות, RTL נכון, חתימה במקום
9. **גיבוי**: ייצוא JSON → מחיקת DB → ייבוא → בדיקה שהכל חזר
10. **בדיקת ביצועים**: טעינת גלריה עם 100+ תמונות גדולות (lazy loading)

### בדיקות ספציפיות
- ✓ RTL נכון בכל המסכים
- ✓ עברית מוצגת נכון ב-PDF (זה האתגר הכי גדול - pdf-lib + fontkit)
- ✓ אפליקציה עובדת ללא אינטרנט
- ✓ נתונים שורדים reload של הדף
- ✓ תמונות נטענות מהר (lazy + thumbnails)
